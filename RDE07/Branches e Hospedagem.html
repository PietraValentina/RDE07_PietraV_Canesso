Branches

São ramificações de um projeto de software ou documento, com objetivo de desenvolver em paralelo novas funcionalidades, corrigir problemas e alterar código.

  • Branch master ou main:

    braço principal de um projeto em um repositório.

  • Branch secundária:

    braço paralelos de um projeto em um repositório.

Passos para criar uma branch dentro de um Projeto.

  1. Branch master ou main:

    Braço principal de um projeto em um repositório.

  2. Branch secundária:

    Braço paralelos de um projeto em um repositório.

    3. Passos para criar uma branch dentro de um Projeto.

    1. Clicar para alterar a branch. Canto superior da tela.
    2. Adicionar "New Branch".
    3. Nomear a nova Branch > Create Branch.
    4. Publicar a Branch no repositório origem.
    5. Para inciar a programar em paralalo na nova Branch, direciona a nova aba no Visual Studio Code.
    6. Após finalizar as alterações, retorna ao GitHub Desktop e da um commit.
    7. Clica em "Push Origem" para enviar ao repositório origem.
    8. Após realizar todas as devidas alterações e invia-las para o repositório origem, é necessário adicionar a Branch secundária na Branch Main.
      Branch > Merge into Current Branch.

    Hospedagem

    O GitHub oferece hospedagem gratuita de sites. Basta criar um ropositóio publico e gerar uma URL.

    Passos para gerar uma URL:

    1. "Logar" na conta do GitHub site.
    2. Abrir o repositório.
    3. Clicar em Settings > Pages.
    4. Selecionar qual Brance deseja.
    5. Salvar e copiar a URL gerada.

    Caso mesmo após postado o site, seja necessário realizar mudanças, é preciso fazer todos os passos novamente.

    OBSERVAÇÃO: sempre deixar no modo publico para funcionar normalmente.
